## Conventions & good practices
- **Respect NDAs.** Keep current work generic — name employers/products only where
  already approved (e.g. Desk5), never client-confidential details.
- Keep the split between **professional experience** and **student/side projects**.
- Factual accuracy: job titles, tech badges, and project descriptions must match
  reality; don't invent specifics (hardware, deployments) without a source.
- Verify every badge/image/link renders (shields.io URLs, project links) before publishing.
